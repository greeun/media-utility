## 1. 개요

### 1.1 목적
이미지 편집기에 밝기(Brightness) 조절 기능을 추가하여 사용자가 업로드한 이미지의 밝기를 실시간으로 조절하고 적용할 수 있도록 한다.

### 1.2 배경
현재 이미지 편집기는 자르기, 회전, 뒤집기, 크기 변경, 최적화 기능을 제공하고 있으나, 밝기/명도 조절 같은 색상 보정 기능이 없다. 밝기 조절은 사진 편집의 가장 기본적인 기능 중 하나로, 사용자 경험을 크게 향상시킬 수 있다.

### 1.3 목표
- 슬라이더 UI로 밝기 값을 -100 ~ +100 범위에서 조절
- Canvas API를 활용한 클라이언트 사이드 처리 (서버 불필요)
- 실시간 미리보기 제공
- 기존 편집 기능(자르기, 회전 등)과 일관된 UX 유지

## 3. 기술 분석

### 3.1 구현 방식: Canvas ImageData 픽셀 조작

Canvas API의 `getImageData()`로 픽셀 데이터를 가져온 뒤, 각 픽셀의 RGB 값에 밝기 오프셋을 더하는 방식이다.

```
밝기 적용 공식:
  offset = brightnessValue * 2.55  (0~255 범위로 변환)
  R' = clamp(R + offset, 0, 255)
  G' = clamp(G + offset, 0, 255)
  B' = clamp(B + offset, 0, 255)
  A' = A (투명도는 유지)
```

**선택 이유:**
- 프로젝트의 기존 패턴(Canvas 기반 처리)과 일치
- 외부 라이브러리 의존성 없음
- 브라우저 호환성 우수

**대안 검토:**
- CSS `filter: brightness()` → 실시간 미리보기에는 좋으나 최종 결과를 Blob으로 내보내기 어려움
- WebGL → 성능은 우수하나 이 프로젝트에는 과도한 복잡성

### 3.2 실시간 미리보기 전략

슬라이더 조작 시마다 Canvas 처리를 실행하면 성능 이슈가 발생할 수 있다.
CSS `filter: brightness()`로 미리보기를 보여주고, "적용" 시에만 Canvas 픽셀 조작을 수행하는 하이브리드 방식을 사용한다.

```
슬라이더 조작 → CSS filter로 즉시 미리보기 (0ms 지연)
적용 버튼 클릭 → Canvas ImageData로 실제 픽셀 변환 → editedBlob 업데이트
```

