## 개요

이 문서는 Media Utility의 **API 테스트, 통합 테스트, E2E 테스트, 비기능 테스트(보안/접근성/성능/i18n)** 시나리오와 핵심 케이스를 정의한다. Part 1(단위/컴포넌트 테스트)의 후속 문서이다.

### 문서 구조

| 섹션 | 범위 | 테스트 수(예상) |
|------|------|-----------------|
| 1. API 테스트 | 6개 엔드포인트 | ~25 케이스 |
| 2. 통합 테스트 | 6개 워크플로우 | ~20 케이스 |
| 3. E2E 테스트 | 16개 페이지 (도구 15 + 파일 뷰어 1) | ~68 케이스 |
| 4. 보안 테스트 | 파일검증/CORS/XSS | ~10 케이스 |
| 5. 접근성 테스트 | 키보드/스크린리더/색상 | ~8 케이스 |
| 6. 성능 테스트 | 처리시간/메모리/로드 | ~8 케이스 |
| 7. i18n 테스트 | 11개 언어/라우팅/RTL | ~10 케이스 |

| **기대 상태** | 400 |

| 항목 | 내용 |
|------|------|
| **사전 조건** | 현재 사용량이 10GB에 근접 |

| **기대 상태** | 507 |

| 항목 | 내용 |

## 3. E2E 테스트

> 모든 E2E 테스트는 Chromium, Firefox, WebKit 3개 브라우저에서 실행한다.
> 각 페이지별 공통 검증: 페이지 로드, 업로드 영역, 설정 UI, 처리/변환, 결과 표시, 다운로드

### 3.1 공통 시나리오 패턴

모든 도구 페이지에 적용되는 공통 패턴:

| 단계 | 검증 항목 |
|------|-----------|
| 페이지 로드 | 200 응답, 제목 표시, 핵심 UI 렌더링 |
| 업로드 영역 | 드래그앤드롭 영역 존재, 파일 선택 버튼, 허용 포맷 안내 |
