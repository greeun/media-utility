## 3. 테스트 케이스

### TC-UNIT-001: adjustBrightness() 휘도만 적용 시 가중치 기반 픽셀 변환

| 항목 | 내용 |
|------|------|
| **우선순위** | Critical |
| **전제조건** | - 4x1 px 테스트 이미지 (RGBA: 100,150,200,255) |
| **테스트 데이터** | `brightness=0, luminance=50` |

| # | 단계 | 예상 결과 |
|---|------|----------|
| 1 | `adjustBrightness(testBlob, 0, 50)` 호출 | Promise<Blob> 반환 |
| 2 | 결과 Blob을 Canvas에 그려 픽셀 데이터 추출 | 픽셀 데이터 접근 가능 |
| 3 | R,G,B 값 검증 | luminance = 0.2126*100 + 0.7152*150 + 0.0722*200 = 142.58<br>adjust = 142.58 * 0.5 = 71.29<br>R' = clamp(100+71.29) = 171<br>G' = clamp(150+71.29) = 221<br>B' = clamp(200+71.29) = 255 (클램핑) |

- 자동화: 가능 (Jest + Canvas mock)
- 관련 요구사항: FR-01, FR-05

| 항목 | 내용 |
|------|------|
| **우선순위** | Critical |
| **전제조건** | - 4x1 px 테스트 이미지 (RGBA: 100,100,100,255) |
| **테스트 데이터** | `brightness=20, luminance=30` |

| # | 단계 | 예상 결과 |
|---|------|----------|
| 1 | `adjustBrightness(testBlob, 20, 30)` 호출 | Promise<Blob> 반환 |
| 2 | 결과 픽셀 데이터 추출 | 접근 가능 |
| 3 | 밝기 오프셋 검증 | brightnessOffset = 20*2.55 = 51<br>R,G,B 각각 100+51 = 151 |
| 4 | 휘도 추가 적용 검증 | L = 0.2126*151 + 0.7152*151 + 0.0722*151 = 151<br>adjust = 151 * 0.3 = 45.3<br>R',G',B' = clamp(151+45.3) = 196 |

- 자동화: 가능
- 관련 요구사항: FR-03

### TC-UNIT-004: 경계값 테스트 - 휘도 극단값

| 항목 | 내용 |
|------|------|
| **우선순위** | High |
| **전제조건** | - 테스트 이미지 (RGBA: 128,128,128,255) |
| **테스트 데이터** | `luminance=-100`, `luminance=0`, `luminance=+100` |

| # | 단계 | 예상 결과 |
|---|------|----------|
| 1 | `adjustBrightness(testBlob, 0, -100)` | luminanceFactor=-1, 픽셀이 어두워짐<br>L=128, adjust=-128<br>RGB = clamp(128-128) = 0 |

| 3 | `adjustBrightness(testBlob, 0, 100)` | luminanceFactor=1, 픽셀이 밝아짐<br>L=128, adjust=128<br>RGB = clamp(128+128) = 255 |
| 4 | 모든 경우 RGB가 0~255 범위 내 | 오버플로/언더플로 없음 |

- 자동화: 가능

### TC-UNIT-005: ITU-R BT.709 가중치 채널별 차등 적용 검증

| 항목 | 내용 |
|------|------|
| **우선순위** | High |
| **전제조건** | - 테스트 이미지 (RGBA: 255,0,0,255) — 순수 빨간색 |
| **테스트 데이터** | `brightness=0, luminance=50` |

| # | 단계 | 예상 결과 |
|---|------|----------|
| 1 | `adjustBrightness(testBlob, 0, 50)` 호출 | 정상 반환 |
| 2 | 순수 빨간색 휘도 계산 검증 | L = 0.2126*255 + 0.7152*0 + 0.0722*0 = 54.21<br>adjust = 54.21 * 0.5 = 27.11 |
| 3 | RGB 결과 확인 | R'=clamp(255+27.11)=255, G'=clamp(0+27.11)=27, B'=clamp(0+27.11)=27 |
| 4 | 순수 녹색(0,255,0)으로 동일 테스트 | L=182.38, adjust=91.19<br>R'=91, G'=255, B'=91 |
| 5 | 빨간색 vs 녹색 결과 비교 | 녹색이 더 큰 adjust 값 → 녹색 가중치(0.7152)가 더 크기 때문 |

- 자동화: 가능
- 비고: 휘도가 밝기와 다른 핵심 차이점 검증
